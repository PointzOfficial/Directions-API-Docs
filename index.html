Pointz Directions API

Introduction

Use our customized Directions API to navigate through cities smoothly on shared e-scooters and e-bikes of a specified brand.

With our API services, you will be able to:

  • Generate e-scooter and e-bike-specific routes that avoid no-ride zones, account for slow streets, and use designated parking areas. This data will be provided to Pointz by the brand.
  • Split the route into three separate legs. The first leg involves walking directions to the brand's vehicle. The second leg provides riding directions from the vehicle's location to the parking area closest to the destination. The final leg provides walking directions from the parking area to the route’s destination.
  • View guided turn-by-turn instructions for each leg of the route.

Authentication

API-Key

To use the Pointz Directions API services, each brand will need a unique Pointz Partner key. This key will be generated from Pointz's backend and assigned to the brand. When sending a request to the Directions API, the key will need to be provided as a header with the following parameter name:

Security Scheme Type: API Key
Header parameter name: Pointz-Partner-Key

Directions API

E-Scooter directions between two points

Calculates an e-scooter route between two points, including any initial and final walking legs to get from a source to a vehicle, or from a vehicle to a destination. This API call can be used for the following use cases:

  • Use any scooter of the specified brand:

    Generate a route that traverses from start to end and navigates to the closest scooter of the brand, brand_id.

  • Use a scooter at a specified location:

    Generate a route that traverses from start to end and navigates to a particular scooter of the brand, brand_id, given by the parameter ride_start_location.

  • Update a route in progress via rerouting:

    Generate rerouting on a current route by sending a request to the Directions API with the same parameters to the original API call, as well as the rider’s current_location and which leg of the route they are on via ride_state.

Successful responses (HTTP code 200) will count as one request to the Directions API.

Authorizations:
API-Key
query Parameters
start
required
Array of numbers <double> [ items <double > ]
Example: start=41.823585,-71.393562

The ‘latitude, longitude’ degrees of the source location in WGS84 format. Only the first 6 decimal places will be used.

end
required
Array of numbers <double> [ items <double > ]
Example: end=41.826769,-71.403238

The ‘latitude, longitude’ degrees of the destination location in WGS84 format. Only the first 6 decimal places will be used.

brand_id
required
string
Example: brand_id=BrandName

The ID for the specified brand of e-scooters.

ride_state
string
Enum: "walking_to_vehicle" "riding" "walking_to_end"
Example: ride_state=walking_to_vehicle

Describes what leg of the route the user is currently in. By default, all routes start with walking_to_vehicle. This parameter can be filled to share the user's current leg for rerouting.

current_location
string
Example: current_location=41.820003,-71.408110

The ‘latitude, longitude’ degrees of the current user location in WGS84 format. Only the first 6 decimal places will be used. Used for rerouting to calculate an alternative route from the original route.

ride_start_location
required
string
Example: ride_start_location=41.826719,-71.399214

The ‘latitude, longitude’ degrees of the location of the vehicle to be used during the walking_to_vehicle leg and the start of the riding legs in WGS84 format. Only the first 6 decimal places will be used.

ride_end_location
required
string
Example: ride_end_location=41.825819,-71.396243

The ‘latitude, longitude’ degrees of the location where the vehicle was dropped off at the end of the riding leg and start of the walking_to_end leg in WGS84 format. Only the first 6 decimal places will be used.

language
string
Example: language=en-US

The language of the turn-by-turn directions based off of a IETF BCP 47 language tag string. By default, “en-US” is used.

start_bearing
integer
Example: start_bearing=37

Clockwise angle from North between 0 and 359. It is used to influence the starting direction to travel in when forming the route.
